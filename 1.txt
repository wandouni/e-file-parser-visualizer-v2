  二、代码层面的小问题

  4. proxy.ts 对每个页面请求都查一次数据库判断 is_banned（性能隐患）
    - 当前对所有非 /login 路由都会执行一次 SELECT is_banned FROM profiles
    - 建议：只在 /admin 和敏感路由检查，普通页面依赖 session token 即可
  5. /api/cases 的 GET 对每个案例各发 2 条 COUNT 查询（N+1 查询）
    - 10 个案例 = 20 次额外查询
    - 建议：改用一条 SQL 用 COUNT + GROUP BY 或 Supabase 的聚合 join 一次取完
  6. Realtime INSERT 时新记录插在列表最前面，但实际应按 sort_order 排
