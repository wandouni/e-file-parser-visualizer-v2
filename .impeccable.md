# Impeccable Design Config

## Project Overview

**E-File Parser & Visualizer v2** — A Next.js 16 full-stack tool for parsing and visualizing Chinese power-grid "E-file" data. Used by electrical grid engineers and data analysts. The UI is entirely in Chinese (Simplified).

## Tech Stack

- **Framework:** Next.js 16 (App Router), React 19
- **Styling:** Pure inline styles (`React.CSSProperties`) — NO Tailwind in component JSX
- **Icons:** Lucide React
- **Charts:** Chart.js + react-chartjs-2
- **Animation:** Motion (v12)
- **Typography:** `'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', sans-serif` (Chinese-first)

## Design Context

### Users

Chinese power-grid engineers and technical analysts. They work in focused, data-heavy sessions — reading tabular E-file data, comparing histories, running visualizations. The interface must be efficient and low-friction. They expect professional tooling, not consumer-grade design.

### Brand Personality

**Professional · Precise · Trustworthy**

The tool should evoke confidence and control. No decoration for decoration's sake. Every visual element earns its place by aiding comprehension or reducing cognitive load.

### Aesthetic Direction

- **Reference:** VS Code / IDE-style — panel-based layout, monospace sensibility, compact information density, clear active/inactive states, subtle but clear visual hierarchy
- **Theme:** Light mode primary (current light main + dark sidebar split is intentional and should be preserved)
- **Anti-reference:** Avoid consumer SaaS aesthetics (rounded card stacks, pastel colors, large whitespace padding, emoji-heavy UI). This is a power tool.

### Design Principles

1. **Density over decoration.** 12px base font is a feature, not a bug. Never pad layouts or increase font sizes just to fill space. Respect engineers' screen real estate.
2. **High contrast is non-negotiable.** All text must meet or exceed WCAG AA contrast ratios. Prefer exceeding them. The user explicitly flagged 高对比度 (high contrast) as a requirement.
3. **Sidebar = command center.** The dark sidebar (`#0f172a`) is the primary navigation surface. Keep it focused, hierarchical, and low-noise. Active states must be immediately obvious.
4. **State clarity first.** Interactive elements (buttons, inputs, rows, tabs) must have unambiguous hover, active, disabled, and focus states. No guessing whether something is clickable.
5. **Chinese typography precision.** PingFang SC / Microsoft YaHei at small sizes requires careful weight and line-height tuning. Prefer `fontWeight: 500–600` for labels; avoid `300/400` at sub-13px sizes.
